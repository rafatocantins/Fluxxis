# Testing & Validation Skill

## Purpose
Write and run comprehensive tests for all system components.

## Capabilities
- Unit tests for components and utilities
- Integration tests for event flow
- E2E tests for smart components
- Performance benchmarks
- Accessibility testing

## When to Use
- Adding tests for new features
- Running verification before commits
- Debugging failing tests
- Setting up CI/CD pipelines

## MCP Integration
- `github` — CI/CD integration, issue management
- `context7` — Latest testing library documentation
- `appcontext` — Visual regression testing

## Test Coverage Targets
| Category | Coverage |
|----------|----------|
| Unit Tests | >90% |
| Integration Tests | >80% |
| E2E Tests | Critical paths only |
| Accessibility | WCAG 2.1 AA |
| Performance | <50ms decision latency |

## Output
- Jest/Vitest test files
- Playwright E2E tests
- Performance benchmarks
- Accessibility audit reports
