# Documentation Skill

## Purpose
Create and maintain clear, comprehensive documentation for the AI Design System.

## Capabilities
- API documentation
- Installation guides
- Usage examples
- Architecture diagrams
- Changelog maintenance

## When to Use
- Documenting new features
- Creating getting started guides
- Writing migration guides
- Updating README

## MCP Integration
- `github` — Repository browsing, issue tracking
- `context7` — Latest framework documentation
- `filesystem` — File structure management

## Documentation Standards
- Clear, concise language
- Code examples for every feature
- TypeScript types in all examples
- Before/after comparisons for changes
- Accessibility considerations

## Output
- Markdown documentation
- TypeScript usage examples
- Architecture diagrams (Mermaid)
- Changelog entries
