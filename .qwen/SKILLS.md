## 📋 Skill Rules

### Rule 1: TypeScript Excellence

**All code must:**
- Use strict TypeScript
- Have proper type definitions
- Avoid `any` type (use `unknown` if needed)
- Follow ESLint rules
- Follow Prettier formatting

### Rule 2: Testing Excellence

**All code must:**
- Have unit tests
- Have integration tests (if applicable)
- Achieve >90% coverage
- Pass all tests before merge

### Rule 3: Performance Excellence

**All code must:**
- Meet performance budgets
- Include performance benchmarks
- Be optimized for production
- Include performance monitoring

### Rule 4: Security Excellence

**All code must:**
- Follow security best practices
- Have no secrets in code
- Use environment variables
- Pass security scans (npm audit)

### Rule 5: Documentation Excellence

**All code must:**
- Have JSDoc comments
- Include usage examples
- Be in task records
- Update changelog

## 🎯 Skill Development

### Skill Levels

| Level | Description | Requirements |
|-------|-------------|--------------|
| **Junior** | Learning skill | Supervised tasks |
| **Mid** | Proficient in skill | Independent tasks |
| **Senior** | Expert in skill | Can teach others |
| **Principal** | Defines best practices | Industry recognition |

### Skill Assessment

**Assessment occurs:**
- Before task assignment
- Monthly for all agents
- After major incidents

**Assessment criteria:**
- Code quality
- Test coverage
- Performance metrics
- Documentation quality
- Ethics compliance
