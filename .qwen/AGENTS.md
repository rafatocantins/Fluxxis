| Agent | Purpose | Priority |
|-------|---------|----------|

## 📋 Agent Rules

### Rule 1: Always Follow the Master Plan

**All agents must align with FLUXXIS-MASTER-PLAN.md.**

- Every task must have a task ID (e.g., "FLX-AG-01")
- No unplanned features
- Phase N must complete before Phase N+1

### Rule 2: Performance Budgets Are Non-Negotiable

**All code must meet performance budgets:**

- Core bundle: <10KB gzipped
- Intent resolution: <10ms
- Morph application: <16ms
- Agent detection: <5ms

### Rule 3: Tests Required

**No code without tests:**

- Unit tests for all functions
- Integration tests for features
- Coverage >90%
- All tests passing

### Rule 4: Document Everything

**All work documented:**

- Task records created
- Changelog updated
- Weekly progress published

### Rule 5: Ethics First

**All code must pass ethics check:**

- No dark patterns
- Privacy by design
- Accessibility WCAG 2.2 AA
- User control preserved

## 🎯 Agent Protocols

### Task Assignment

```
1. Orchestrator receives task from Master Plan
2. Orchestrator assigns to specialist agent
3. Agent executes task
4. Testing Validator validates
5. Documentation Writer documents
6. Orchestrator marks complete
```

### Code Review

```
1. Agent creates PR
2. Testing Validator runs tests
3. Ethics Compliance checks ethics
4. Code Architect reviews architecture
5. Orchestrator approves merge
```

### Incident Response

```
1. Detect incident (P0, P1, P2, P3)
2. Orchestrator assigns to appropriate agent
3. Agent responds within SLA
4. Testing Validator validates fix
5. Documentation Writer updates incident log
```
